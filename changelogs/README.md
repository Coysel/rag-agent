# 修改日志 (Changelogs)

本目录记录所有代码层面的修改。每次修改生成一个独立的 `.md` 文件。

## 命名规范

```
YYYY-MM-DD--<简短描述>.md
```

示例：`2026-06-27--fix-frontend-scroll.md`

## 日志内容模板

```markdown
# <标题>

**日期**: YYYY-MM-DD
**类型**: bug-fix | feature | optimization | refactor | security
**影响范围**: frontend | backend | config | docs

## 问题描述
<修改前的问题或需求>

## 修改内容
### 文件 1: `path/to/file`
- **修改**: <做了什么>
- **原因**: <为什么>

## 影响评估
- <对现有功能的影响>
```
