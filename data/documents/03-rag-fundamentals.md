# RAG (Retrieval-Augmented Generation) 基础

## 什么是 RAG

RAG（检索增强生成）是一种将信息检索与文本生成相结合的技术架构。在 LLM 生成答案之前，先从外部知识库检索相关文档，然后将检索结果作为上下文提供给 LLM。

## RAG 解决的核心问题

1. **知识截止日期限制**：LLM 训练数据有截止日期，RAG 可以检索最新信息
2. **幻觉问题 (Hallucination)**：基于检索到的真实文档生成，大幅减少编造内容
3. **领域知识不足**：可以接入特定领域的专业文档，无需微调模型
4. **可溯源**：每个答案都有引用的来源文档，可验证

## RAG 的基本流程

```
用户问题 → 检索相关文档 → 拼接上下文 → LLM 生成回答
```

1. **索引 (Indexing)**：将文档库切分、生成 Embedding、存入向量数据库
2. **检索 (Retrieval)**：将用户问题转为向量，在向量数据库中搜索相似文档
3. **生成 (Generation)**：将检索到的文档 + 用户问题组成 prompt，让 LLM 生成答案

## RAG vs 标准 LLM

| 特性 | 标准 LLM | RAG |
|------|----------|-----|
| 知识更新 | 依赖训练数据 | 实时检索 |
| 幻觉控制 | 难以控制 | 基于文档，可溯源 |
| 领域适配 | 需要微调 | 换文档库即可 |
| 成本 | 推理成本 | + 检索成本 |
| 可解释性 | 低 | 高（有引用来源） |

## RAG 的局限

- 检索质量决定回答质量上限
- 简单问题 RAG 可能过度检索，增加延迟
- 分块策略影响检索精度和上下文完整性
- 对多步推理问题的支持有限（需要 Agentic RAG）
