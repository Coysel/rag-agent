# RAGAS 评估框架

## 什么是 RAGAS

RAGAS (Retrieval Augmented Generation Assessment) 是一个专门用于评估 RAG 系统的开源框架。它提供了一套标准化的指标，用于量化评估 RAG 系统的检索和生成质量。

## 四个核心指标

### 1. Faithfulness（忠实度）

**衡量什么**：生成的回答是否有检索到的文档作为依据。

**为什么重要**：这是衡量"幻觉"的指标。即使检索到了正确的文档，LLM 也可能在其中添加不存在的信息。

**计算方式**：
- 将回答拆分为原子陈述（atomic statements）
- 逐条检查每个陈述是否能在检索到的文档中找到依据
- Faithfulness = 能找依据的陈述数 / 总陈述数

### 2. Answer Relevance（答案相关性）

**衡量什么**：回答是否切合用户的问题。

**为什么重要**：检索到相关文档后，LLM 仍可能生成偏离问题的回答。

**计算方式**：
- 根据回答反向生成假设问题
- 计算原始问题与假设问题之间的语义相似度
- Relevance = 相似度平均值

### 3. Context Precision（上下文精度）

**衡量什么**：检索到的文档中，有多少是与问题真正相关的。

**为什么重要**：衡量"搜到的有没有用"。精度低意味着检索到了大量噪声文档，浪费了上下文窗口。

**计算方式**：
- 判断每个检索到的文档是否与问题相关
- Precision = 相关文档数 / 检索到的总文档数

### 4. Context Recall（上下文召回）

**衡量什么**：标准答案中需要的信息，有多少在检索到的文档中找到了。

**为什么重要**：衡量"有用的检没检索到"。召回低意味着遗漏了关键信息，回答可能不完整。

## 四个指标的关系

| | 检索到了 | 没检索到 |
|---|---|---|
| **相关文档** | ✓ Precision ↑, Recall ↑ | ✗ Recall ↓ |
| **不相关文档** | ✗ Precision ↓ | ✓（无影响） |

- **Precision 和 Recall 相互制约**：提高一个可能降低另一个
- **Faithfulness 依赖检索质量**：检索不到相关文档时 LLM 容易瞎编
- **Relevance 依赖生成质量**：即使检索完美，LLM 也可能答偏

## 为什么单看一个指标不够

| 场景 | Faithfulness | Relevance | Precision | Recall | 诊断 |
|------|-------------|-----------|-----------|--------|------|
| 完美 | 高 | 高 | 高 | 高 | ✓ |
| 检索偏题 | 高 | 低 | 低 | 低 | 检索策略需改进 |
| 生成跑偏 | 高 | 低 | 高 | 高 | Prompt 需优化 |
| 检索不足 | 低 | 中 | 高 | 低 | 增加检索范围 |
| 幻觉严重 | 低 | 高 | 高 | 高 | 加强引用约束 |

四个指标一起看才能全面诊断 RAG 系统的问题所在。

## 面试要点

RAGAS 是区分"调 API"和"真做工程"的分水岭。面试官会追问：
- 四个指标分别衡量什么？
- 某个指标低说明什么问题？如何改进？
- 你做的系统中各指标是多少？如何提升的？
